# TopGear - ניהול מוסך מקומי

אפליקציית שולחן עבודה, 100% אופליין, לניהול יומן עבודות, מלאי חלקים ומסירות במוסך.

## התקנה (Windows)

1. עברו לעמוד [Releases](../../releases) בריפו זה.
2. הורידו את הגרסה האחרונה — קובץ בסיומת `.msi` או `.exe`.
3. הריצו את ההתקנה. אם Windows SmartScreen מציג התרעה (האפליקציה אינה חתומה דיגיטלית), לחצו "More info" → "Run anyway".
4. הפעילו את **TopGear** מתפריט ההתחלה.

כל הנתונים נשמרים מקומית במחשב, ללא חיבור לאינטרנט וללא שרת.

## הפעלה ללא התקנה

ניתן גם לפתוח את `index.html` בדפדפן שולחני מודרני (Chrome / Edge) ולעבוד באותה צורה. הנתונים יישמרו ב-IndexedDB של הדפדפן.

## בנייה מקומית מהקוד

דרישות:

- [Rust](https://www.rust-lang.org/tools/install) (יציב)
- [Node.js](https://nodejs.org/) 20+
- WebView2 — מותקן כברירת מחדל ב-Windows 11

צעדים:

```bash
# יצירת אייקונים מהלוגו (פעם אחת)
npx @tauri-apps/cli@latest icon assets/topgear.png

# מצב פיתוח עם hot-reload
npx @tauri-apps/cli@latest dev

# בניית installer ל-Windows
npx @tauri-apps/cli@latest build
```

ה-installer ייווצר ב-`src-tauri/target/release/bundle/`.

## פרסום גרסה חדשה

ה-CI (`.github/workflows/release.yml`) בונה אוטומטית עבור Windows כשמושך תג `vX.Y.Z`:

```bash
# עדכנו את "version" ב-src-tauri/tauri.conf.json לפני התיוג
git tag v0.1.1
git push origin v0.1.1
```

לאחר סיום ה-build, התקנה (`.msi`/`.exe`) תופיע כ-**Draft Release** בעמוד Releases — בדקו, ולחצו **Publish release**.

## עדכונים אוטומטיים

TopGear משתמש ב-[Tauri Updater Plugin](https://v2.tauri.app/plugin/updater/). כשהלקוח פותח את האפליקציה, היא ניגשת בשקט ל-Releases של GitHub, ומציעה התקנה אם פורסמה גרסה חדשה.

### הגדרה חד-פעמית

1. **ייצרו זוג מפתחות חתימה** (מאומת על-ידי האפליקציה לפני שתתקין עדכון):

   ```powershell
   npx @tauri-apps/cli@latest signer generate -w topgear-updater.key
   ```

   - `topgear-updater.key` — **מפתח פרטי**. אל תעלו לגיט. שמרו במקום מאובטח.
   - `topgear-updater.key.pub` — מפתח ציבורי.

2. **הדביקו את המפתח הציבורי ב-`src-tauri/tauri.conf.json`** — שדה `plugins.updater.pubkey`. החליפו `REPLACE_WITH_PUBLIC_KEY` בערך המודפס בקובץ `.pub`.

3. **עדכנו את ה-endpoint** ב-`plugins.updater.endpoints` עם שם המשתמש/הריפו שלכם, למשל:

   ```json
   "endpoints": [
     "https://github.com/morad/topgear/releases/latest/download/latest.json"
   ]
   ```

4. **הוסיפו ל-GitHub Secrets** (Settings → Secrets and variables → Actions):

   - `TAURI_SIGNING_PRIVATE_KEY` — התוכן המלא של `topgear-updater.key` (בצעו `cat` או פתחו כטקסט).
   - `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` — הסיסמה שבחרתם בעת היצירה (אם לא בחרתם — סוד ריק).

### זרימת העדכון בצד הלקוח

- האפליקציה בודקת עדכון 3 שניות לאחר העלייה (`checkForUpdates` ב-`app.js`).
- אם נמצאה גרסה חדשה — מוצגת בקשת אישור בעברית עם מספר הגרסה.
- בלחיצה על "אישור": ההורדה, האימות (מול המפתח הציבורי), ההחלפה והאתחול קורים אוטומטית.
- במצב דפדפן רגיל (`index.html` ללא Tauri) — הבדיקה לא פועלת ואין השפעה.

## שמירת נתוני הלקוח בעדכון

הנתונים של הלקוח **לא נמחקים בעדכון**. הסיבות:

- ה-IndexedDB נשמר תחת `%LOCALAPPDATA%\com.topgear.garage\EBWebView\` (מנותק מקובץ ה-EXE).
- ה-installer של Tauri (`MSI` + `NSIS`) מבצע **upgrade-in-place** — מחליף את הבינארי בלבד, ללא מחיקת תיקיית המשתמש.
- `installMode: currentUser` ב-`tauri.conf.json` מבטיח שהתקנה לכל משתמש אינדיבידואלי לא תתנגש עם משתמשים אחרים.
- מזהה האפליקציה (`identifier: com.topgear.garage`) **אינו משתנה בין גרסאות** — אם הוא ישתנה, WebView2 תפנה לתיקייה חדשה והנתונים יאבדו. אסור לשנות אותו לאחר ההשקה.

לבדיקת שלמות הנתונים אחרי עדכון — הריצו "ייצוא JSON מלא" לפני עדכון משמעותי. אם משהו השתבש, ייבאו את הקובץ חזרה ממסך "גיבוי וייצוא".

## אחסון נתונים

הנתונים נשמרים מקומית באמצעות IndexedDB:

- במצב התקנה (`.msi`) — בתיקיית WebView2 של האפליקציה תחת `%LOCALAPPDATA%\com.topgear.garage\EBWebView\`.
- במצב דפדפן — בפרופיל הדפדפן.

טבלאות:

- `jobs` - עבודות ותיקונים
- `inventory` - מלאי חלקים
- `categories` - קטגוריות חלקים מותאמות אישית

המחירים של חלקים נשמרים גם בתוך כל עבודה כצילום מצב, כדי ששינוי מחיר עתידי במלאי לא ישנה עבודות היסטוריות.

לגיבוי / העברה בין מחשבים — השתמשו ב"ייצוא JSON מלא" ממסך **גיבוי וייצוא**.

## יכולות עיקריות

- יומן עבודה בעברית וב-RTL מלא
- חישוב אוטומטי של יום בשבוע, סה"כ ורווח
- קטלוג מלאי עם מק"ט ייחודי, כמות ומחירים
- סינון מלאי לפי קטגוריה כולל קטגוריות מותאמות אישית
- יצירת חלק חדש ישירות מתוך מסך הוספת עבודה
- בחירת חלקים מתוך כרטיסיות לפי קטגוריה, כולל חיפוש, מלאי ומחיר גלויים
- הוספת חלקים לעבודה מתוך הכרטיס עצמו, עם תצוגת מלאי שמתעדכנת בזמן אמת
- מסך **מסירות צפויות** עם סינון לפי טווח (היום / השבוע / החודש / באיחור)
- ניכוי מלאי אוטומטי בעת שמירת עבודה
- החזרת מלאי בעת מחיקת עבודה
- באנר אנליטיקה להיום, השבוע, החודש או הכל
- ייצוא JSON מלא (כולל קטגוריות מותאמות)
- ייצוא CSV ליומן ולמלאי
- ייבוא מגיבוי JSON
- טעינת נתוני דוגמה ממסך `גיבוי וייצוא`
